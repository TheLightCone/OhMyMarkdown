# Windows平台Markdown编辑器开发需求文档

## 一、项目概述

**项目名称**：Windows平台增强型Markdown编辑器
**项目目标**：开发一款Windows平台Markdown编辑器，以Typora为功能基准，在界面布局、文件管理、搜索能力、扩展性四个维度进行增强。
**目标平台**：Windows 10/11

---

## 二、核心基准：Typora全功能复刻

编辑器必须完整实现Typora的所有核心功能，作为产品基础体验：

### 2.1 实时预览编辑模式

- 所见即所得（WYSIWYG）编辑体验
- 输入Markdown语法即时渲染为富文本样式
- 支持源码模式与预览模式切换

### 2.2 Markdown语法支持

- **标准语法**：标题、段落、列表、引用、代码块、链接、图片、表格、分隔线等
- **扩展语法**：
  - 任务列表（`- [ ]` / `- [x]`）
  - 脚注
  - 上标/下标
  - 高亮标记
  - 目录（TOC）
- **图表支持**：
  - Mermaid 流程图、时序图、甘特图等
  - PlantUML
  - 数学公式（LaTeX / MathJax）

### 2.3 主题与样式

- 内置多种主题（Light / Dark / 经典等）
- 支持自定义CSS主题
- 代码块语法高亮主题可配置

### 2.4 导出功能

- 导出为 PDF（支持页眉页脚、页码、封面）
- 导出为 HTML（含/不含样式）
- 导出为 Word (.docx)
- 导出为图片（PNG/JPG）
- 导出为纯文本

### 2.5 基础文件树

- 文件夹层级浏览
- 文件新建/重命名/删除
- 文件拖拽排序

## 三、界面布局重构

### 3.1 整体布局

```
+-------------------------------------------------------------+
|  顶部菜单栏（文件 | 编辑 | 视图 | 插件 | 帮助）              |
+----------+--------------------------------+---------------+
|          |                                |               |
|  左侧    |        中央编辑区              |    右侧       |
|  边栏    |        (实时预览编辑)          |    边栏       |
|          |                                |               |
|  文件树  |                                |   大纲/目录   |
|          |                                |   导航        |
|          |                                |               |
+----------+--------------------------------+---------------+
|  底部状态栏（行号/列号 | 字数统计 | 当前编码 | 当前主题）   |
+-------------------------------------------------------------+
```

### 3.2 左侧边栏：文件树


| 功能     | 说明                              |
| ------ | ------------------------------- |
| 文件夹层级  | 树形结构展示工作区文件夹及文件                 |
| 文件操作   | 右键菜单：新建文件/文件夹、重命名、复制、移动、删除、批量操作 |
| 文件状态标识 | 未保存（●）、已归档（📁）、进行中（📝）等状态图标     |
| 拖拽支持   | 文件/文件夹拖拽移动、外部文件拖入导入             |
| 快速定位   | 支持按文件名首字母快速跳转                   |
| 工作区切换  | 支持多个工作区（项目文件夹）切换                |


### 3.3 右侧边栏：大纲/目录导航


| 功能     | 说明                      |
| ------ | ----------------------- |
| 标题层级   | 自动提取当前文档所有标题（H1-H6）     |
| 层级折叠   | 支持展开/折叠各级标题             |
| 点击跳转   | 点击大纲项自动滚动定位到对应位置        |
| 当前位置高亮 | 编辑区滚动时，大纲自动高亮当前所在章节     |
| 大纲过滤   | 支持按标题级别过滤显示（如仅显示H1-H2）  |
| 大纲拖拽   | 支持拖拽标题调整文档结构（调整标题顺序/层级） |


### 3.4 底部状态栏

- 当前行号/列号
- 文档总字数/选中字数
- 当前文件编码（UTF-8 / GBK 等）
- 当前应用主题
- 自动保存状态指示

## 四、增强文件管理系统

### 4.1 基础文件操作


| 操作   | 功能描述                 |
| ---- | -------------------- |
| 新建   | 新建Markdown文件、新建文件夹   |
| 重命名  | 文件/文件夹重命名，自动更新内部链接引用 |
| 移动   | 拖拽或剪切/粘贴移动文件/文件夹     |
| 复制   | 复制文件/文件夹             |
| 删除   | 删除至回收站，支持彻底删除        |
| 批量操作 | 多选文件后批量重命名、批量移动、批量删除 |
| 导入   | 从外部导入Markdown文件/文件夹  |
| 收藏   | 收藏常用文件/文件夹，快速访问      |


### 4.2 版本历史


| 功能     | 说明                      |
| ------ | ----------------------- |
| 自动保存版本 | 每次手动保存或定时自动保存时创建版本快照    |
| 版本列表   | 按时间线展示所有历史版本            |
| 版本对比   | 双栏对比查看两个版本差异（类Git diff） |
| 版本回滚   | 一键回滚到指定历史版本             |
| 版本备注   | 支持为重要版本添加备注标签           |
| 版本清理   | 自动清理过期版本（可配置保留策略）       |
| 存储位置   | 版本数据存储于应用数据目录，不污染原文件    |


### 4.3 自动备份


| 功能      | 说明                  |
| ------- | ------------------- |
| 定时备份    | 可配置备份间隔（默认5分钟）      |
| 备份路径    | 支持自定义本地备份目录         |
| 异常恢复    | 程序异常关闭后重启，提示恢复未保存内容 |
| 备份策略    | 支持保留最近N个备份或保留N天内的备份 |
| 云备份（可选） | 预留接口，支持后续对接云存储备份    |


### 4.4 文件生命周期管理


| 状态  | 说明                     |
| --- | ---------------------- |
| 进行中 | 正在编辑的活跃文件              |
| 已完成 | 编辑完成的文件，可标记为完成         |
| 已归档 | 不再活跃但需要保留的文件，归档后从主视图折叠 |
| 已删除 | 软删除状态，保留在回收站中          |


- 支持按状态筛选文件树显示
- 支持文件标签系统（自定义标签分类）
- 支持项目级文件夹管理（一个项目包含多个关联文件夹）

## 五、增强搜索系统

### 5.1 搜索面板

```
+------------------------------------------+
|  🔍 搜索关键词                           |
+------------------------------------------+
|  查找范围：[当前文件夹 ▼]                 |
|  查找类型：[文件内容 ▼]                   |
|  ☑ 区分大小写  ☐ 全字匹配  ☐ 正则表达式 |
+------------------------------------------+
|  搜索结果（10条）                         |
|  ├── 📄 文档A.md                        |
|  │   └── ...关键词上下文... [第15行]     |
|  ├── 📄 文档B.md                        |
|  │   └── ...关键词上下文... [第42行]     |
|  └── ...                               |
+------------------------------------------+
```

### 5.2 搜索维度


| 维度       | 选项                                 |
| -------- | ---------------------------------- |
| **查找范围** | 当前文件夹 / 指定文件夹 / 全局工作区 / 当前打开文件     |
| **查找类型** | 文件名 / 文件内容 / 文件标签 / 文件状态           |
| **匹配规则** | 区分大小写（开关）/ 全字匹配 / 正则表达式支持          |
| **文件过滤** | 按文件类型（.md / .txt / 全部）、修改时间、文件大小过滤 |


### 5.3 搜索结果展示

- 文件名 + 匹配内容摘要（关键词高亮）+ 行号/位置
- 点击结果直接跳转定位到对应位置
- 支持搜索结果导出
- 搜索历史记录

### 5.4 高级搜索

- 支持搜索语法（如 `title:关键词`、`tag:重要`、`status:进行中`）
- 支持保存常用搜索条件为快捷搜索

## 七、技术栈建议


| 层级        | 技术选型                             | 说明                  |
| --------- | -------------------------------- | ------------------- |
| **桌面框架**  | Electron + React / Tauri + React | 跨平台桌面应用框架           |
| **编辑器内核** | CodeMirror 6 / ProseMirror       | 高性能富文本/ Markdown 编辑 |
| **渲染引擎**  | Markdown-it + 自定义插件              | Markdown解析与渲染       |
| **状态管理**  | Zustand / Redux                  | 应用状态管理              |
| **样式方案**  | Tailwind CSS + CSS Modules       | UI样式                |
| **文件存储**  | 本地文件系统 + SQLite                  | 文件操作与元数据存储          |
| **搜索索引**  | SQLite FTS5 / FlexSearch         | 全文搜索索引              |
| **版本控制**  | 自定义快照系统                          | 基于文件差异的版本历史         |
| **打包工具**  | Vite / esbuild                   | 构建与打包               |

## 八、非功能需求

### 8.1 性能要求

- 启动时间 < 3秒
- 打开10MB文件无明显卡顿
- 实时预览渲染延迟 < 100ms
- 搜索1000个文件响应时间 < 2秒

### 8.2 兼容性

- 支持 Windows 10 (1903+) / Windows 11
- 支持标准Markdown文件（.md / .markdown）
- 兼容Typora主题格式

### 8.3 安全性

- 插件沙箱隔离
- 自动备份防止数据丢失
- 敏感操作（删除、回滚）需二次确认

### 8.4 可扩展性

- 插件系统预留充足API接口
- 主题系统支持深度自定义
- 配置项支持导入/导出
